================================================================================
  ETIQUETAS SUGERIDAS PARA GITHUB ISSUES
================================================================================
  tipo:bug              — Error en funcionalidad existente
  tipo:mejora           — Mejora o refactor
  tipo:feature           — Nueva funcionalidad
  tipo:documentacion     — Documentacion
  tipo:seguridad         — Parche de seguridad
  prioridad:alta         — Bloqueante o critico
  prioridad:media        — Importante pero no urgente
  prioridad:baja         — Nice to have
  area:backend           — Java / API
  area:frontend          — Web SPA
  area:javafx            — Desktop GUI
  area:bd                — Base de datos
  area:devops            — Despliegue / CI
  estado:bloqueado       — Depende de otro issue
  estado:en-progreso     — En desarrollo
  estado:revision        — Esperando revision


================================================================================
  FLUJO DE TRABAJO RECOMENDADO (Git Flow simplificado)
================================================================================
  Rama principal:
    main          — Codigo estable en produccion

  Ramas de desarrollo:
    develop       — Integracion de cambios
    feature/*     — Nuevas funcionalidades (ej. feature/pagos-parciales)
    fix/*         — Correccion de bugs (ej. fix/qr-toctou)
    docs/*        — Documentacion (ej. docs/modelo-relacional)

  Proceso:
    1. Crear issue en GitHub Projects
    2. Crear rama desde develop: feature/nombre-corto
    3. Desarrollar y hacer commits individuales por cambio
    4. Abrir Pull Request a develop
    5. Revisar y mergear
    6. Al alcanzar un hito, mergear develop a main


================================================================================
  COMMITS RECOMENDADOS POR CAMBIO (no agrupar)
================================================================================
  Cada cambio debe tener su propio commit con mensaje descriptivo:

  Formato: tipo(area): mensaje en espanol imperativo

  Ejemplos:
    feat(bd): agregar tabla ALERTAS_PAGO con FK a CUOTAS_ARRIENDO
    feat(api): implementar endpoint POST /api/pagos con validacion
    feat(frontend): agregar columna Arrendatario en tabla contratos
    fix(backend): corregir TOCTOU en marcado de QR como usado
    fix(frontend): mostrar fallback embed para PDF en escaner QR
    refactor(dao): extraer metodo findByContrato en CuotaArriendoDAO
    docs: agregar modelo relacional con 18 tablas y 28 relaciones
